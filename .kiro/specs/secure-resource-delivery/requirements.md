# Requirements Document

## Introduction

The Colonel's Academy currently stores PDF course materials (study guides, resource packs, reference documents) in Bunny CDN/Storage with raw public URLs saved in the `Lesson.pdfUrl` database field. These URLs are passed directly to the frontend and rendered as plain `<a href>` links in `ResourceLessonStage`, meaning any user who obtains a URL can access the content indefinitely — regardless of enrollment status or payment.

This feature replaces that approach with a secure, signed-URL delivery pipeline. The Fastify API will act as the access-control gatekeeper: it verifies enrollment, generates short-lived signed tokens for Bunny CDN, and serves PDFs through an in-platform viewer with watermarking and copy-protection. The raw CDN URL is never exposed to the browser.

Critically, the PDF viewer renders **inline in the same position as the VideoPlayer** inside the classroom lesson viewer — not as a separate link or modal. When a lesson has `contentType=PDF` or `learningMode=RESOURCE`, `LessonStageRenderer` routes to the new `SecurePdfViewer` component instead of `ResourceLessonStage`. The existing `ResourceLessonStage` "Open Resource" link behaviour is replaced entirely.

## Glossary

- **Resource_Delivery_Service**: The Fastify API module responsible for verifying access and issuing signed resource tokens.
- **Signed_URL**: A time-limited, HMAC-authenticated URL generated by the Resource_Delivery_Service that grants temporary read access to a specific file on Bunny CDN.
- **SecurePdfViewer**: The Next.js client-side PDF viewer component that renders inline where the VideoPlayer currently renders, using `react-pdf` to render pages with a canvas watermark overlay.
- **Watermark**: A semi-transparent overlay rendered on each PDF page inside the SecurePdfViewer, containing the authenticated user's display name, email, and a timestamp.
- **Enrollment_Guard**: The existing `assertLessonAccess` function in `apps/api/src/lib/access-guard.ts` that verifies active enrollment and phase/prerequisite gates.
- **Token_TTL**: The configurable time-to-live for a Signed_URL, defaulting to 60 seconds.
- **Bunny_CDN**: The content delivery network and storage provider already configured in the platform for hosting course assets.
- **pdfUrl**: The existing `Lesson.pdfUrl` database field, which stores the internal Bunny Storage path or full CDN URL for a lesson's PDF resource.
- **Classroom_Page**: The Next.js page at `/classroom/[slug]` where enrolled students view lesson content.
- **LessonStageRenderer**: The component at `apps/web/src/components/classroom/LessonStageRenderer.tsx` that routes a lesson to the appropriate stage component.

## Requirements

### Requirement 1: Signed URL Generation Endpoint

**User Story:** As an enrolled student, I want to view a lesson's PDF resource inline in the classroom, so that I can study the material without leaving the platform.

#### Acceptance Criteria

1. WHEN an authenticated user requests a resource token for a lesson via `GET /v1/learning/resources/:lessonId/token`, THE Resource_Delivery_Service SHALL verify the user's enrollment and lesson access using the Enrollment_Guard before issuing any token.
2. IF the Enrollment_Guard denies access, THEN THE Resource_Delivery_Service SHALL return HTTP 403 with a descriptive error message and SHALL NOT issue a Signed_URL.
3. WHEN access is confirmed, THE Resource_Delivery_Service SHALL generate a Signed_URL for the lesson's Bunny CDN asset using HMAC-SHA256 with the Bunny CDN token authentication key.
4. THE Resource_Delivery_Service SHALL set the Signed_URL expiry to the configured Token_TTL value (default 60 seconds).
5. THE Resource_Delivery_Service SHALL scope each Signed_URL to the specific file path of the requested lesson, so that the token cannot be reused to access other assets.
6. THE Resource_Delivery_Service SHALL return the Signed_URL to the authenticated client over HTTPS only.
7. IF a lesson has no associated pdfUrl, THEN THE Resource_Delivery_Service SHALL return HTTP 404 with the message "No resource attached to this lesson."
8. THE Resource_Delivery_Service SHALL log each token issuance with the userId, lessonId, and expiry timestamp for audit purposes.

---

### Requirement 2: Raw URL Suppression

**User Story:** As a platform operator, I want raw CDN URLs to never reach the browser, so that content cannot be shared or accessed outside the platform.

#### Acceptance Criteria

1. THE Resource_Delivery_Service SHALL omit the `pdfUrl` field from all lesson responses returned by the catalog and classroom endpoints when the requesting user is not an ADMIN or DS role.
2. WHEN a lesson has a PDF resource and the user is enrolled, THE Resource_Delivery_Service SHALL include a boolean flag `hasResource: true` in the lesson response in place of the raw URL.
3. THE SecurePdfViewer SHALL obtain the Signed_URL by calling the token endpoint at the moment the viewer mounts, not from the lesson payload.
4. WHERE the user holds an ADMIN or DS role, THE Resource_Delivery_Service SHALL continue to include the raw `pdfUrl` in lesson responses to support content management workflows.

---

### Requirement 3: Inline Secure PDF Viewer

**User Story:** As an enrolled student, I want to read PDF materials inline in the classroom where the video player normally appears, so that the experience is seamless and the file cannot be downloaded or shared easily.

#### Acceptance Criteria

1. WHEN a lesson has `contentType=PDF` or `learningMode=RESOURCE`, THE LessonStageRenderer SHALL render the SecurePdfViewer component in place of the VideoPlayer — in the same position and layout slot where the video player currently renders.
2. THE SecurePdfViewer SHALL render PDF pages using `react-pdf` without exposing the source URL in the browser's address bar or DOM.
3. THE SecurePdfViewer SHALL disable the browser's native PDF download button by serving the PDF through a proxy response with `Content-Disposition: inline` and `X-Content-Type-Options: nosniff`.
4. THE SecurePdfViewer SHALL apply CSS rules to disable text selection and right-click context menus on the PDF rendering area.
5. WHEN the Signed_URL expires during an active viewing session, THE SecurePdfViewer SHALL automatically request a new token and reload the current page without interrupting the user's reading position.
6. THE SecurePdfViewer SHALL display a loading state while the Signed_URL is being fetched and while PDF pages are rendering.
7. IF the token request fails due to a network error, THEN THE SecurePdfViewer SHALL display a user-friendly error message and a retry button.
8. THE SecurePdfViewer SHALL be responsive and render correctly on viewport widths from 375px to 1920px.
9. THE existing `ResourceLessonStage` component with its "Open Resource" link SHALL be replaced by the SecurePdfViewer for lessons with a PDF resource.

---

### Requirement 4: User Watermarking

**User Story:** As a platform operator, I want each PDF view to display the student's identity, so that leaked screenshots or recordings can be traced back to the source.

#### Acceptance Criteria

1. THE SecurePdfViewer SHALL overlay a watermark on each rendered PDF page containing the authenticated user's display name, email address, and the current date in ISO 8601 format.
2. THE SecurePdfViewer SHALL render the watermark at a fixed opacity between 0.08 and 0.15 so that it is visible but does not obstruct the content.
3. THE SecurePdfViewer SHALL position the watermark diagonally across the centre of each page at a 30-degree angle.
4. THE SecurePdfViewer SHALL render the watermark using a canvas overlay so that it cannot be removed by inspecting or modifying the PDF source.
5. IF the authenticated user's display name or email is unavailable, THEN THE SecurePdfViewer SHALL substitute the user's Firebase UID in the watermark.

---

### Requirement 5: Proxy Endpoint (Download Prevention)

**User Story:** As a platform operator, I want PDF bytes to be streamed through the API rather than fetched directly from CDN, so that the CDN URL is never visible in browser network traffic.

#### Acceptance Criteria

1. THE Resource_Delivery_Service SHALL expose a proxy endpoint `GET /v1/learning/resources/:lessonId/view` that re-validates enrollment, fetches the PDF bytes from Bunny CDN, and streams them to the client.
2. THE Resource_Delivery_Service SHALL set the response header `Content-Disposition: inline; filename="resource.pdf"` on all proxied PDF responses.
3. THE Resource_Delivery_Service SHALL set `Cache-Control: no-store, no-cache` on all proxied PDF responses to prevent browser caching of the raw bytes.
4. THE Resource_Delivery_Service SHALL re-validate the user's enrollment on every proxy request, not only on token issuance.
5. IF the upstream Bunny CDN request fails or returns a non-200 status, THEN THE Resource_Delivery_Service SHALL return HTTP 502 with the message "Resource temporarily unavailable."
6. THE Resource_Delivery_Service SHALL limit proxied response size to 50 MB per request to prevent abuse.

---

### Requirement 6: Access Audit Logging

**User Story:** As a platform operator, I want a record of every resource access attempt, so that I can detect sharing or abuse patterns.

#### Acceptance Criteria

1. THE Resource_Delivery_Service SHALL record each successful resource access in the database with the userId, lessonId, courseId, IP address, user agent, and access timestamp.
2. THE Resource_Delivery_Service SHALL record each denied resource access attempt with the userId (if authenticated), lessonId, denial reason, and timestamp.
3. THE Resource_Delivery_Service SHALL NOT store the Signed_URL itself in the audit log.
4. WHEN a single userId accesses the same resource more than 20 times within a 60-minute window, THE Resource_Delivery_Service SHALL emit a warning-level log entry flagging the userId and lessonId for review.
5. THE Resource_Delivery_Service SHALL expose an admin-only endpoint that returns the audit log entries for a given lessonId, paginated in reverse chronological order.

---

### Requirement 7: LessonStageRenderer Integration

**User Story:** As an enrolled student, I want the classroom to show the PDF viewer inline where the video normally plays, so that the experience feels native to the platform.

#### Acceptance Criteria

1. WHEN a lesson's `contentType` is `PDF` or `learningMode` is `RESOURCE`, THE LessonStageRenderer SHALL route to the SecurePdfViewer instead of `ResourceLessonStage`.
2. THE SecurePdfViewer SHALL render within the same layout container as the VideoPlayer — inside the `<div className="bg-black">` wrapper in the Classroom_Page.
3. WHEN a lesson's `hasResource` flag is `false` or absent, THE SecurePdfViewer SHALL display a "Resource Pending" disabled state consistent with the existing pattern.
4. WHEN the user marks the lesson complete, THE SecurePdfViewer SHALL allow the mark-complete action to proceed independently of the PDF viewing state.
5. THE SecurePdfViewer SHALL display the lesson title and synopsis, consistent with the existing stage component patterns.
